Core capabilities

Everything you'd expect from an OS — delivered through a chat thread.

💬

Session-based conversations

Context persists across turns. CuriOS remembers who you are and where you are in every flow.

🧭

Guided onboarding flows

Auth and provider configuration happen through natural dialogue, not complex setup screens.

🔌

Provider abstraction

Swap between LLM backends at any time without touching a single line of OS logic.

🎯

Intent handling

Free-form messages are mapped to a controlled set of deterministic system actions.

📜

Message history

Full conversation logs stored and retrievable, with automatic secret redaction built in.

🗺️

Extensible by design

A plugin and tool system is on the roadmap — notes, files, browser automation, and more.

How it works

Four clean steps from your message to the next reply.

  1. 01

    You send a message

    Anything from "help" to "change provider" — no syntax required.

  2. 02

    Session context is resolved

    CuriOS looks up who you are, your current flow state, and what's already configured.

  3. 03

    Message is routed through a state machine

    Not authenticated → auth flow. No AI provider → provider setup. Otherwise → ready state with full intent handling.

  4. 04

    Reply returned & state persisted

    CuriOS responds, then saves updated state and message history for the next turn.
